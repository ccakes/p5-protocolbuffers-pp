# CLAUDE.md

## Project Overview

ProtocolBuffers::PP is a pure-Perl Protocol Buffers implementation. It provides binary wire format encoding/decoding, canonical JSON (ProtoJSON) mapping, and a `protoc` code generator plugin (`protoc-gen-perl`).

## Key Commands

```bash
# Run unit tests (225 tests)
prove -r t/unit/

# Run the official protobuf conformance suite
./conformance-runner

# Generate Perl code from .proto files
protoc --perl_out=lib/ --plugin=protoc-gen-perl=script/protoc-gen-perl \
    -I /path/to/includes file.proto

# Install dependencies
cpanm --installdeps .
```

## Architecture

### Runtime (lib/ProtocolBuffers/PP/)

- **Encode.pm / Decode.pm** — Binary protobuf encoding/decoding. Descriptor-driven: iterate field descriptors by number, encode/decode based on type. Oneofs tracked via `_oneof_case` hash in messages.
- **Wire/** — Low-level wire format: Varint.pm (variable-length ints), ZigZag.pm (signed int encoding), Bytes.pm (fixed-width, length-delimited), Tags.pm (field tag encode/decode).
- **JSON/Print.pm** — Message-to-JSON. WKT dispatch table for special types. Uses `FloatLiteral` (Math::BigFloat subclass) for precise float formatting through JSON::PP.
- **JSON/Parse.pm** — JSON-to-message. WKT dispatch table. Uses `B::svref_2object` SV flags to distinguish JSON numbers from JSON strings (critical for Struct/Value).
- **Types.pm** — Constants for all protobuf field types (TYPE_DOUBLE, TYPE_STRING, etc.) and labels.
- **Util.pm** — `default_value_for_type`, `is_packable`, helper functions.
- **Error.pm** — Structured errors with category (decode, json, etc.) that stringify for eval/die pattern.
- **Bootstrap.pm** — Hand-built descriptors for `google.protobuf.compiler.CodeGeneratorRequest/Response` so the generator can decode its own input without generated code.
- **Generator.pm** — Generates Perl packages from parsed proto descriptors. Produces message classes with `__DESCRIPTOR__` methods, accessor methods with oneof semantics, and enum classes.
- **Timestamp.pm, Duration.pm, FieldMask.pm, Any.pm** — Well-Known Type string conversion helpers.

### Generated Code (lib/Google/Protobuf/, lib/Conformance/)

Generated by `protoc-gen-perl`. Each message class has:
- `__DESCRIPTOR__` method returning a hash with field definitions, field lookups by name/json_name, oneofs, and resolved `message_descriptor` references.
- Accessor methods for each field (with oneof clearing logic for oneof members).
- Enum classes with `name_for`/`value_for` lookups.

### Message Representation

Messages are plain Perl hashes (not blessed objects during encode/decode). Special keys:
- `_oneof_case` — `{ oneof_index => active_field_number }` tracks which oneof field is set
- `_unknown_fields` — Raw bytes of unrecognized fields, preserved for round-tripping

### Conformance Harness (script/perl-conformance-harness)

Pipe-based protocol: reads 4-byte LE length prefix + protobuf ConformanceRequest, writes 4-byte LE length prefix + protobuf ConformanceResponse. Handles JSON and protobuf input/output formats. Maintains a type registry for Any resolution.

## Important Patterns

- **Perl string semantics matter**: `utf8::decode()` on protobuf TYPE_STRING output marks bytes as character strings for JSON::PP; `utf8::encode()` converts back before wire encoding. JSON::PP with `utf8(1)` expects raw UTF-8 bytes, not character strings.
- **Perl falsy values**: Use `//` (defined-or) not `||` for protobuf payloads — the byte `\x30` (ASCII "0") is a valid payload but falsy in Perl. Same issue with empty-string field defaults.
- **JSON number vs string distinction**: JSON::PP sets `SVp_IOK`/`SVp_NOK` flags on numbers and `SVp_POK` on strings. The `_is_json_number()` function in Parse.pm uses `B::svref_2object` to inspect these flags. This is essential for google.protobuf.Value (Struct) to correctly map JSON `"5678"` to string_value vs `5678` to number_value.
- **Proto3 default omission**: Singular non-oneof fields at their default value are omitted from both binary encoding and JSON output.
- **Map fields**: Encoded as repeated MapEntry messages. Descriptors have `map_entry` with `key_type`, `value_type`, `value_message_descriptor`, `value_enum_class`.

## External Dependencies

- **protoc** — Only needed for code generation, not at runtime
- **conformance_test_runner** — Built from protobuf source.
- **Perl modules** — JSON::PP, MIME::Base64, Math::BigInt, POSIX (all core or near-core). Test deps: Test::More, Test::Exception.
